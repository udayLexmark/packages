Routing/Mapping Screen
Use the Routing/Mapping screen to:
Set up content based routing or mapping for IDocs before processing them in the SAP Adapter or before sending them to an SAP system.
Register an inbound or outbound service. To do so, click Register New User Exit Service. The Types Screen appears.
Unregister an inbound or outbound service. To do so, click Unregister User Exit Service. The Types Screen appears.

The Content based Routing/Mapping Settings for IDocs section contains the following information:
 
Column
Description/Action
Default handling for all IDocs
Displays the Inbound Service and Outbound Service to be used for default handling of IDocs.
A default handling service (inbound or outbound) is executed for all message types. You can disable a previously registered default service by selecting the special value $none instead.
Message type dependent settings
For each message type, select the inbound and/or outbound service.
If there are services selected in the this section, they will be executed for the corresponding message type in addition to the default service. You can disable a previously registered message type dependent service by selecting the special value $none instead. The SAP Adapter is shipped with the following sample routing/mapping services:
For all incoming IDocs with an ORDERS message type, the sample.sap.idoc.Mappings:orders service will be executed. The service may alter the IDoc as well as the routing parameters for the IDoc.
For IDocs going to the SAP system with ORDRSP message type, the sample.sap.idoc.Mappings:ordrsp service will be executed. This service can alter the IDoc by adding or removing fields.
The Content based Routing Settings for XML section contains the following information:
 
Column
Description/Action
Handling for all incoming XML documents
Specifies the service to be used for to route all XML documents to the routing listener.
After making your changes, click Save to save the routing/mapping information.
