Configure Listener Type Screen (RFC Listener)
Use the Configure Listener Type screen to configure an RFC listener.
1
Complete the following fields (leave all other fields at their default values):
 
Field
Description/Action
Package
The package in which to create the listener. You must create the package using the Developer before you can specify it using this parameter. For general information about creating packages, see the webMethods Developer User’s Guide.
Note: Configure the listener in a user-defined package rather than in the adapter’s package. See the webMethods SAP Adapter User’s Guide for other important considerations when creating packages for the SAP Adapter.
Folder Name
The folder in which to create the listener.
Listener Name
The name of the new listener.
Program ID
The Program ID that you specified when creating the corresponding RFC destination on the SAP system. This field is case sensitive.
Gateway Host
Select the Gateway Host for accessing your SAP system. This must be exactly the same parameter as you chose for the corresponding RFC destination in the SAP system.
Gateway Service
The Gateway Service. This corresponds to your SAP system number. If your SAP system number is “01” then your gateway service is “sapgw01.”
Again, this must be exactly the same parameter as you chose for the corresponding RFC destination in the SAP system.
Number of Threads
Select the number of simultaneous incoming RFCs that this listener can handle.
Repository Server
Select the outbound connection alias used as a repository for function interfaces and structure definitions of inbound calls. This way it is possible to use RFCs even if they are not defined in the calling system.
SNC Enabled
Yes/No. Specifies whether this server should use SNC. Default: No.
SNC Quality of Service
Select the SNC Quality of service:
Use global build-in default settings
Plain text, but authorization
Each data packet will be integrity protected
Each data packet will be privacy protected
Use maximum available security
SNC Name
Your own SNC name if you do not want to use the default SNC name. This is the name you chose when generating a PSE.
Authorization Service
The name of the service called by the RFC listener to check for authorization. To provide application specific authorization handling the user might provide a service here that implements specification pub.sap.listener:listenerAuthorizationCheck. This service returns “Granted” in field “access” if the request should be accepted.
If no service is specified, access will be granted.
If an exception happens during execution of this service or a different string than “Granted” will be returned, access will be denied.
See the webMethods SAP Adapter User’s Guide for information on about the pub.sap.listenerAuthorizationCheck service specification.
RFC Trace
On/Off. Specifies whether you want RFC tracing enabled or disabled. For a productive system, select Off. When you select On, the SAP Adapter collects tracing in rfc*.trc files in directory webMethods_directory\Integration Server\.
For detailed information about logging and changing the directory for the trace files, see the webMethods SAP Adapter User’s Guide.
Log transaction status
On/Off. If set to Off, the processing logs will not be saved, although a transaction will be created (or maintained), and the transaction can be monitored later on in the transaction list.
Tip! Setting this switch to Off reduces the amount of disk space needed and the time it takes to log the transaction status. It still allows you to check the current transaction status on the SAP Adapter.
Store message body
On/Off. If set to Off, the message body of the incoming document will not be stored to disk, although a transaction will be created (or maintained), and the transaction can be monitored later on in the transaction list.
Tip! Setting this switch to Off reduces the amount of disk space needed and the time it takes to persist the message body. It still allows you to track the message status on the SAP Adapter.
Important! An RFC listener must be able to login to the SAP system automatically when it starts up. Additionally, if there are no metadata yet for called function modules in the cache, there must be a log in option to the SAP system assigned as repository, as well. Otherwise, it will fail to start up or return errors when receiving incoming RFCs.
2
Select Save Listener to commit these settings.
The RFC listener you created appears on the adapter’s Listeners screen and in the Developer.
Edit Listener Screen (RFC Listener)
Use the Edit Listener screen to update the properties for an existing RFC listener. You cannot edit an RFC listener if it is enabled.
For descriptions of the RFC listener’s parameters, see Configure Listener Type Screen (RFC Listener).
When you are done making changes to the RFC listener properties, click Save Changes.
To return to the Listeners screen, click Return to SAP Adapter Listeners.
To change the order in which the SAP Adapter will process the notifications, click Edit Notification Order.
View Listener Screen (RFC Listener)
Use the View Listener screen to view properties for an existing RFC listener.
For descriptions of the RFC listener’s parameters, see Configure Listener Type Screen (RFC Listener).
To return to the Listeners screen, click Return to SAP Adapter Listeners.
To view the order in which the SAP Adapter will process the notifications, click View Notification Order.
Copy Listener Screen (RFC Listener)
Use the Copy Listener screen to create a new RFC listener by copying the parameters for an existing RFC listener. You cannot copy the routing listener.
Name the RFC listener and edit any parameters you want to change for the new listener. Click Save Changes. For descriptions of the listener’s parameters, see Configure Listener Type Screen (RFC Listener).
Edit Notification Order Screen (RFC Listener)
Use the Edit Notification Order screen to change the order in which the SAP Adapter will process the notifications. Use the Up and Down buttons to arrange the notification order.
For better processing results, arrange your notifications from ascending to descending order starting with the most detailed notifications to the least detailed notifications.
Click Save Changes to save the notification order of the listener. To return to the Edit Listener page, click Return to Edit Listener.
View Notification Order Screen (RFC Listener)
Use the View Notification Order screen to view the order in which the SAP Adapter will process the notifications.
To return to the View Listener page, click Return to View Listener.
