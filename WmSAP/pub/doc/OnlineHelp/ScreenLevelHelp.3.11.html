Edit Listener Screen (Routing Listener)
Use the Edit Listener screen to update the properties for the routing listener. You cannot edit the routing listener if it is enabled.
You can edit the following properties:
 
Field
Description/Action
Listener Type
Specifies the type of listener.
Package Name
The namespace node package in which in which the listener resides.
Retry Limit
The number of times that the system should attempt to start the listener if the initial attempt fails.
The default is 5. The value 0 means that the system makes a single attempt.
Retry Backoff Timeout
The number of seconds the system should wait between each attempt to start the listener. The default is 10.
This field is irrelevant if Retry Limit is 0.
Log transaction status
On/Off. If set to Off, the processing logs will not be saved, although a transaction will be created (or maintained), and the transaction can be monitored later on in the transaction list.
Tip! Setting this switch to Off reduces the amount of disk space needed and the time it takes to log the transaction status. It still allows you to check the current transaction status on the SAP Adapter.
Store message body
On/Off. If set to Off, the message body of the incoming document will not be stored to disk, although a transaction will be created (or maintained), and the transaction can be monitored later on in the transaction list.
Tip! Setting this switch to Off reduces the amount of disk space needed and the time it takes to persist the message body. It still allows you to track the message status on the SAP Adapter.
When you are done making changes to the routing listener properties, click Save Changes.
To return to the Listeners screen, click Return to SAP Adapter Listeners.
To change the order in which the SAP Adapter will process the notifications, click Edit Notification Order.

Edit Notification Order Screen (Routing Listener)
Use the Edit Notification Order screen to change the order in which the SAP Adapter will process the notifications. Use the Up and Down buttons to arrange the notification order.
For better processing results, arrange your notifications from ascending to descending order starting with the most detailed notifications to the least detailed notifications.
Click Save Changes to save the notification order of the listener. To return to the Edit Listener page, click Return to Edit Listener.
