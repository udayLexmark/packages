Edit Password Expiration Settings Screen
Screen Heading: Security > User Management > Password Expiration Settings > Edit
Use the Edit Password Expiration Settings screen to edit the criteria for password expiry. You also use this screen to specify whether to enable the expiry settings.
To open the Edit Password Expiration Settings screen, go to the User Management Screen, click Password Expiration Settings, and then click Edit Password Expiration Settings.
Click Return to User Management to return to the list of groups and users that are currently defined on your server.
Click Return to Password Expiration Settings to view the password expiry settings.
Edit Password Expiration Settings
Use this section to specify the criteria for password expiration. You also use this screen to specify whether to enable the expiry settings.
Enabled
Specifies whether to enable the password expiry settings. This option is disabled by default. Select Enabled to enable the password expiry settings.
Note: If this option is enabled, then upon save all the passwords that a user had set before the expiration interval are considered expired and users must reset them.
Expiration Interval
Specifies the number of days after which a password will expire, if not changed. The value should be a non-zero integer. Default is 90 days.
Note: Upon save, when this option is enabled, any password that a user had set before the expiration interval are considered expired and the user must reset them. For example, if you changed your password 10 days ago and now, the Administrator changes the Expiration Interval to 5 days, then your password has expired and needs to be reset.
Expiration Notice Email Addresses
Specifies the list of email addresses to which Integration Server sends an email notification informing that the user password is about to expire or has already expired. If you leave this field blank, then Integration Server uses the email address specified in the Internal Email field of Email Notification section on Resources Screen ( Settings > Resources).
Note: Integration Server uses the SMTP server and port details specified in the Email Notification section on Resource Settings Screen (Settings > Resources).
Expiration Email Reminders
Specifies the number of days prior to password expiry that Integration Server should start sending the reminder emails for password reset. The emails are sent daily until the password is either updated or the expiration interval is changed. Set the value to 0 to prevent Integration Server from sending the reminder emails. Default is 3.
Note: Integration Server continues to send reminder emails until the user resets the password, even after the password has expired. Reminder emails are sent for expired passwords, even if you set the value to 0.
Applies to Users
Specifies the list of users to whom the expiration interval setting is applicable. Use the and buttons to move user names on or off this list.
Remaining Users
Specifies the list of users to whom the expiration interval setting is not applicable. Use the and buttons to move user names on or off this list.
Save Changes
Applies the changes you have specified in the Password Expiration Settings. Your changes take effect immediately.
