MQTT Trigger Management Screen
Screen Heading: Settings > Messaging > MQTT Trigger Management
Use the MQTT Trigger Management screen to view the MQTT triggers defined on Integration Server.
To open the MQTT Trigger Management screen, click Messaging under the Settings menu on the Navigation panel. On the Settings > Messaging screen, click MQTT Trigger Management.
Click Return to Message to return to the Messaging screen.
Click View MQTT Settings to vies the MQTT connection aliases.
Click Refresh Page to refresh the page.
MQTT Trigger Controls
Trigger Name
Fully qualified name of the MQTT trigger.
State
Indicates whether the trigger is enabled, disabled, or suspended. Click the trigger state to access the Edit State for Individual MQTT Trigger Screen which you can use to enable, disable, or suspend the MQTT trigger.
edit all
Click to access the Edit State for All JMS Triggers Screen in which you can enable, disable, or suspend all the MQTT triggers on the Integration Server.
Status
The status of the MQTT trigger, which can be “Running” or “Not Running (reason)” where reason identifies why the rigger is not running, such as “Not Running (Connection Alias Not Running)”.
Connection Alias Name
Name of the MQTT connection alias used to retrieve messages fro the trigger.
Topic
Topic on the MQTT server from which the trigger retrieves messages.
Queue Capacity
Maximum number of messages that can be in the local trigger queue awaiting processing.
Current Queue Count
Current number of messages in the local trigger queue.
Maximum Threads
Maximum number of server threads that can process messages concurrently for this MQTT trigger.
Current Threads
Current number of server threads processing messages for this MQTT trigger.
