Edit webMethods Messaging Trigger Properties Screen
Screen Heading: Settings > Messaging > webMethods Messaging Trigger Management > triggerName > Edit Properties
To open the webMethods Messaging Trigger Management screen, click Messaging under the Settings menu on the Navigation panel.
On the Messaging Screen, click webMethods Messaging Trigger Management. On the webMethods Messaging Trigger Management Screen, in the Trigger Name column, click the trigger name whose details you want to edit. On the webMethods Messaging Trigger Detail Screen, click Edit Properties.
Click Return to webMethods Messaging Trigger to return to the previous screen.
Properties
Queue Capacity
The maximum number of documents that the trigger queue can contain.
Note: The Queue Capacity can be set for webMethods messaging triggers that receive documents from Broker only. You cannot use Integration Server Administrator to set queue capacity for webMethods messaging triggers that receive documents from Universal Messaging.
Queue Refill Level
The number of unprocessed documents that must remain in this trigger queue before Integration Server retrieves more documents for the queue from Broker. The Queue Refill Level value must be less than or equal to the Queue Capacity value.
Note: The Queue Refill Level applies to webMethods messaging triggers that receive documents from Broker only. webMethods messaging triggers that receive documents from Universal Messaging do not have a refill level property. The Queue Refill Level field displays N/A for triggers that receive documents from Universal Messaging.
Max Execution Threads
The maximum number of documents that Integration Server can process concurrently. You can only specify a maximum execution threads value for concurrent triggers. This field displays N/A for serial triggers.
Apply Change Across Cluster
Select this check box if you want to apply the property changes for this trigger to all the Integration Servers in a cluster.
Note: The Apply Change Across Cluster check box appears only if the current Integration Server belongs to a properly configured cluster and is configured to synchronize trigger changes across the cluster.
Save Changes
Saves your changes to the webMethods messaging trigger properties.
