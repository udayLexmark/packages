Edit Enhanced XML Parsing Screen
Screen Heading: Settings > Enhanced XML Parsing > Edit
Use the Edit Enhanced XML Parsing screen to configure the default settings for the enhanced XML parser and to indicate if the enhanced XML parser uses BigMemory.
To open the Enhanced XML Parsing screen, go to the Enhanced XML Parsing Screen and click Edit Enhanced XML Parsing Settings.
Click Return to Enhanced XML Parsing Settings to return to the current enhanced XML parsing settings.
Enhanced Parser Settings
Default partition size
Specifies the size, measured in bytes, of the partitions on the heap where the enhanced XML parser stores parsed document information. Specify a suffix of “k” to indicate kilobytes or “m” to indicate megabytes. For example, 10k or 10m. The default is 20k.
Caching
Maximum heap allocation for all documents combined
Maximum amount of heap space that the parser can allocate to process documents concurrently. Specify a suffix of “k” to indicate kilobytes, “m” to indicate megabytes, “g” to indicate gigabytes, and “%” to indicate a percentage of the heap space. For example, 10k, 10m, 10g, or 10%. The default is 40% of the heap space.
Maximum heap allocation for any single document
Maximum amount of heap space that the parser can allocate to process a single document. Specify a suffix of “k” to indicate kilobytes, “m” to indicate megabytes, “g” to indicate gigabytes, and “%” to indicate a percentage of the heap space. For example, 10k, 10m, 10g, or 10%. The default is 20% of the heap space.
BigMemory Caching
Maximum BigMemory allocation
Maximum amount of off-heap memory, measured in bytes, to allocate to BigMemory. Specify a suffix of “k” to indicate kilobytes, “m” to indicate megabytes, or “g” to indicate gigabytes. For example, 10k, 10m, or 10g. The default is 200m.
Save Changes
Saves the specified settings. These settings will take effect immediately.
