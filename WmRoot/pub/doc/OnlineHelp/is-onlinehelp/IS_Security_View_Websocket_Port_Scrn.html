View WebSocket Port Details Screen
Screen Heading: Security > Ports > View WebSocket Port Details
Use the View WebSocket Port Details screen to view or specify the settings for an WebSocket port.
To open the View WebSocket Port Details Screen, click Ports under the Security menu on the Navigation panel.
Click Return to Ports to return to the list of ports that are currently defined on the server.
Click Edit WebSocket Port Configuration to edit general configuration information about the WebSocket port that are currently defined on the server.
Click WebSocket Endpoints to view active sessions for WebSocket server endpoints. You can access this screen if active sessions are available for WebSocket server endpoint.
WebSocket Listener Configuration
Use this section to view general configuration information about this port.
Port
Indicates the number of the port to which you want to assign an WebSocket listener.
Alias
An alias for the port.
Description (optional)
A description of the port.
Package Name
Indicates the name of the package that you have associated with this port.
Bind Address (optional)
Optional. Indicates the IP address to which to bind this port.
Backlog
Indicates the number of requests that can remain in the queue for an enabled port before Integration Server begins rejecting requests. The default is 200. The maximum value is 65535.
Threadpool Configuration
Use this section to view the minimum and maximum number of threads that are reserved for requests.
Threadpool Min
Indicates the minimum number of threads for this private threadpool. The default is 1.
Threadpool Max
Indicates the maximum number of threads for this private threadpool. The default is 5.
Thread Priority
Indicates the Java thread priority. The default is 5.
WebSocket Policy
Indicates the policy details configuration for WebSocket connections.
Idle Timeout(ms)
Indicates the maximum idle time for client connections with the handler. If the specified idle time is exceeded, the connection is torn down.
Security Configuration
Indicates the way you want the server to authenticate clients that submit requests through this port.
