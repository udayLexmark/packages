View Truststore Alias Screen
Screen Heading: Security > Keystore > View Truststore Alias
Use the View Truststore Alias screen to view the properties of a specific truststore alias. A truststore contains the trusted root of the certificate or signing authority (CA). The truststore alias is a user-assigned identifier for the truststore, that can be easily referenced from within Integration Server.
To open the View Truststore Alias screen, click Keystore under the Security menu on the Navigation panel. Select the name of a truststore alias to view its properties.
Click Return to Keystore to return to the list of keystore aliases that are currently defined on the server.
Click Edit Truststore Alias to edit the properties of the truststore alias currently displayed.
Truststore Properties
Alias
Displays the user-specified identifier for the keystore or truststore. Clicking on the truststore alias displays its properties.
Description (optional)
Displays a user-specified description of the alias.
Type
Displays the file format of the keystore or truststore file. The default is JKS. Other truststore types can be made available by loading additional security providers, or by setting the watt.security.TrustStoreTypes server configuration property.
Provider
Displays the provider that is used for the truststore type. The default is the provider shipped with the JVM (Oracle, IBM, or others). If your Hardware Security Module (HSM) device is not supported by the default provider, you can configure a different provider to support truststore file types.
Note: Integration Server only supports the JKS file format for truststores.
Location
Displays the location of the truststore file on the server. This can be the full pathname, or a relative path in relation to the Integration Server.
Certificate Aliases
Displays a list of all truststore certificate alias names defined on Integration Server.
