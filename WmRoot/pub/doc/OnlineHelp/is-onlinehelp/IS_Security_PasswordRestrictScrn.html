Password Restrictions Screen
Screen Heading: Security > User Management > Password Restrictions
Use the Password Restrictions screen to view criteria for setting passwords.
To open the Password Restrictions screen, go to the User Management Screen and click Password Restrictions.
Click Return to User Management to return to the list of groups and users that are currently defined on your server.
Click Edit Password Restrictions to change the password settings.
Password Restrictions
Use this section to view the criteria for setting passwords.
Enable Password Change
Specifies whether non-Administrator users can change their passwords. If you select No, users who are not members of the Administrators ACL must have a system administrator change the passwords for them. The default is Yes.
Password Enforcement Mode
Specifies whether Administrator users are allowed to choose passwords that are not impacted by the password restrictions setting. When set to Lax, Administrator users can choose passwords that do not conform to the password restrictions setting. When set to Strict, Administrator users must choose passwords that follow the password restriction settings. For non-administrators the restrictions are always enforced. The default is Lax.
Minimum Password Length
Specifies the minimum number of total characters a password must contain. The default is 8.
Maximum Password Length
Specifies the maximum number of total characters a password can contain. Maximum number of characters that a password can have is 128. The default is 64 characters.
Minimum Number of Uppercase Characters
Specifies the minimum number of uppercase alphabetic characters the password must contain. The default is 0.
Minimum Number of Lowercase Characters
Specifies the minimum number of lowercase alphabetic characters the password must contain. The default is 0.
Minimum Number of Digits
Specifies the minimum number of numeric digits the password must contain. The default is 0.
Minimum Number of Special Characters (neither alphabetic nor digits)
Specifies the minimum number of special characters, such as asterisk (*), period (.), question mark (?), and ampersand (&), the password must contain. The default is 0.
Maximum Number of Identical Characters in a Row
Specifies the maximum number of continuous identical characters the password can contain. The default is 3.
Number of Old Passwords to Remember (per user)
Specifies the maximum number of previously set passwords that Integration Server remembers for a user (excluding the current password). A new password cannot match any of the passwords stored in memory. The default is 0.
