Server Log Screen
Screen Heading: Logs > Server
Use the Server Log screen to view the contents of the server log (serveryymmdd.log). The server log contains general information about server operations (for example, the loading of packages). User-defined processes that run on the server can also log arbitrary events in the server log. For information about changing the display of this log, see webMethods Audit Logging Guide.
To open the Server Log screen, click Server under the Logs menu on the Navigation panel.
Log display controls
Ascending sequence
View entries in the log from the oldest to the most current.
Descending sequence
View entries in the log from the most current to the oldest.
First line to display
Specifies the first entry you want to be displayed in the log, based on the Log Sequence number in the first column. The default is 0, which displays all lines in the log.
Number of entries to display
By default, the Integration Server Administrator displays the 35 most recent entries in the server log. You can change the number of entries displayed.
Refresh
Click this button to update the display after changing the log display controls.
Tip: If you have Administrator privileges, you can change the default refresh interval for log entries (the default is 90 seconds). Go to the Settings > Extended page and modify the watt.server.log.refreshInterval.
Server Log Entries as of mm-dd hh:mm:ss
This log does not have a top row with headings for each column. The descriptions below match the order of the columns in the log.
Log Sequence
Number that shows the sequence in which the log entry was written to the log.
Timestamp
Date and time the entry was written to the log.
Facility Information
Code that describes the facility or component that added the log message and the log message identifier. For example, the code ISS 0014 0002 C breaks down as follows:
String
Description
ISS
Three-character abbreviation of the facility or component that issued the log message
0014
Four-digit facility code
0002
Message number
C
Severity code of the log message. Possible severity codes are:
C
Critical
D
Debug
E
Error
Vn
Verbose (n is the verbosity level)
W
Warning
Log Message
Message that indicates the action that the server performed.
